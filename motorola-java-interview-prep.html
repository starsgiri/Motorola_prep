4 Pillars of OOP

Most asked — know all 4 with examples

▾
Q1What are the 4 pillars of OOP? Explain each.
1. Encapsulation: Binding data + methods into a single unit (class). Use private fields + public getters/setters. Protects data integrity.

2. Inheritance: A child class acquires properties of parent class using extends. Promotes code reuse.

3. Polymorphism: One interface, many forms. Compile-time (method overloading) and Runtime (method overriding).

4. Abstraction: Hiding implementation details, exposing only what's necessary. Achieved via abstract classes and interfaces. 
// Encapsulation
class Employee {
  private double salary;
  public double getSalary() { return salary; }
  public void setSalary(double s) { this.salary = s; }
}
Q2What is the difference between method overloading and overriding?
Overloading (Compile-time polymorphism): Same method name, different parameters in the SAME class.

Overriding (Runtime polymorphism): Subclass provides specific implementation of a parent class method. Same signature.

Key rule: You can't override static, final, or private methods. 
// Overloading
int add(int a, int b) { return a+b; }
double add(double a, double b) { return a+b; }

// Overriding
class Animal { void sound() { System.out.println("..."); } }
class Dog extends Animal {
  @Override void sound() { System.out.println("Woof"); }
}
Q3Interface vs Abstract Class — when to use which?
Abstract Class: Use when classes share common state/code. Can have constructors, instance variables, concrete methods. A class can extend only ONE abstract class.

Interface: Use for defining a contract/capability. All methods are implicitly public abstract (Java 8+ allows default/static methods). A class can implement MULTIPLE interfaces.

Rule of thumb: "IS-A" → abstract class. "CAN-DO" → interface.
Dog IS-A Animal (abstract class). Dog CAN-DO Swimmable (interface).
Q4What is the use of the super keyword?
super refers to the immediate parent class. Used to:
1. Call parent class constructor: super() — must be first statement
2. Access parent class methods: super.methodName()
3. Access parent class fields if hidden by child 
class Vehicle {
  String type = "Vehicle";
  Vehicle(String t) { this.type = t; }
}
class Car extends Vehicle {
  Car() {
    super("Car"); // calls Vehicle constructor
    System.out.println(super.type); // parent field
  }
}
Q5Can we achieve multiple inheritance in Java? How?
Java does NOT support multiple inheritance via classes (to avoid the diamond problem). However, it IS supported via interfaces — a class can implement multiple interfaces.

Java 8+ allows default methods in interfaces. If two interfaces have the same default method, the implementing class must override it. 
interface Flyable { default void move() { ... } }
interface Swimmable { default void move() { ... } }

class Duck implements Flyable, Swimmable {
  @Override
  public void move() { Flyable.super.move(); } // must resolve
}

Java Fundamentals

String, static, final, constructors

▾
Q6Why is String immutable in Java?
String objects are stored in the String Constant Pool. If strings were mutable, changing one reference would affect all others pointing to the same object. Immutability ensures:
1. Thread safety (no synchronization needed)
2. Security (e.g., class names, DB connection strings)
3. Caching of hashCode works correctly

Mutable alternatives: StringBuilder (not thread-safe, faster) and StringBuffer (thread-safe).
Q7What is the difference between == and .equals()?
== compares references (memory addresses). Two different String objects with same content return false.

.equals() compares the actual content of objects. Always use .equals() for String comparison. 
String a = new String("hello");
String b = new String("hello");
a == b;       // false (different objects)
a.equals(b);  // true (same content)
Q8What does the static keyword mean?
static members belong to the CLASS, not to instances. They're shared across all objects.

- Static variable: Single copy shared by all instances
- Static method: Can be called without creating an object. Cannot access non-static members directly
- Static block: Runs once when class is loaded
- Static nested class: Doesn't need outer class instance
Q9Explain final keyword with all use cases.
final variable: Value cannot be changed (constant). Must be initialized at declaration or in constructor.

final method: Cannot be overridden by subclasses.

final class: Cannot be extended (e.g., String, Integer).

Blank final: A final field not initialized at declaration must be initialized in ALL constructors.
Q10What is a constructor? Rules and types?
A constructor is a special method called when an object is created. Same name as class, no return type.

Default constructor: Provided by compiler if no constructor defined.
Parameterized constructor: Takes arguments.
Copy constructor: Takes same class object as argument.

Constructor chaining: this() calls another constructor in same class. super() calls parent constructor.

List & Set

ArrayList, LinkedList, HashSet, TreeSet

▾
Q11ArrayList vs LinkedList — key differences?
ArrayList: Backed by a dynamic array. O(1) random access. O(n) insertion/deletion in middle (shifting). Best for frequent reads.

LinkedList: Doubly linked list. O(n) for access. O(1) insert/delete at known position. Also implements Deque. Best for frequent add/remove operations.

Interview tip: Mention that ArrayList has a default capacity of 10 and grows by 50% when full.
Q12HashSet vs TreeSet vs LinkedHashSet?
HashSet: No order, no duplicates, O(1) operations. Uses HashMap internally.

TreeSet: Sorted (natural or custom Comparator), no duplicates, O(log n) operations. Uses Red-Black tree.

LinkedHashSet: Maintains insertion order, no duplicates, O(1) operations.

All three use hashCode() and equals() for uniqueness.
Q13What happens if you don't override hashCode() and equals()?
Default equals() uses == (reference comparison). Default hashCode() is based on memory address.

If you override equals() but NOT hashCode(), two logically equal objects may go to different hash buckets — violating the contract. Always override BOTH.

Contract: If a.equals(b) is true, then a.hashCode() == b.hashCode() must also be true.

Map Interface

HashMap, TreeMap, LinkedHashMap

▾
Q14How does HashMap work internally?
HashMap uses an array of buckets (Node[]). Key's hashCode() is computed, then compressed using index = hash & (n-1) to find the bucket.

Collision: Multiple keys at same bucket → stored as LinkedList (Java 8+: converted to Red-Black Tree if chain length > 8).

Load factor: Default 0.75. When 75% full, HashMap resizes (doubles) and rehashes.

Java 8 improvement: Treeification of bins for O(log n) worst case vs O(n) before.
Q15HashMap vs Hashtable vs ConcurrentHashMap?
HashMap: Not thread-safe, allows one null key, fast.

Hashtable: Thread-safe (synchronized methods), no null keys, legacy class — avoid.

ConcurrentHashMap: Thread-safe with segment-level locking (Java 7) or CAS operations (Java 8+). Better performance than Hashtable. No null keys or values. Use this in multi-threaded environments.
Q16How do you iterate over a Map?
Map<String, Integer> map = new HashMap<>();

// Method 1: entrySet (most efficient)
for (Map.Entry<String, Integer> e : map.entrySet()) {
  System.out.println(e.getKey() + " = " + e.getValue());
}

// Method 2: Java 8 forEach + lambda
map.forEach((k, v) -> System.out.println(k + "=" + v));

// Method 3: keySet (less efficient — double lookup)
for (String key : map.keySet()) {
  System.out.println(map.get(key));
}

Lambda & Streams (Java 8)

Big plus for this role — mention it!

Q17What is a Lambda expression? Give an example.
A lambda is an anonymous function — it has no name, can be passed as an argument. Enables functional programming in Java. Works with functional interfaces (interfaces with exactly one abstract method). 
// Before Java 8
Comparator<String> c = new Comparator<String>() {
  public int compare(String a, String b) { return a.compareTo(b); }
};

// With Lambda
Comparator<String> c = (a, b) -> a.compareTo(b);

// Sorting a list
List<String> names = Arrays.asList("Zara","Alice","Bob");
names.sort((a, b) -> a.compareTo(b));
names.sort(String::compareTo); // method reference
Q18What are functional interfaces? Name a few built-in ones.
An interface with exactly ONE abstract method. Annotated with @FunctionalInterface.

Key built-ins in java.util.function:
- Predicate<T> → takes T, returns boolean (test())
- Function<T,R> → takes T, returns R (apply())
- Consumer<T> → takes T, returns void (accept())
- Supplier<T> → no input, returns T (get())
- Runnable, Comparator, Callable are also functional interfaces

Thread Basics

Lifecycle, creation, states

▾
Q19What are the two ways to create a thread in Java?
1. Extend Thread class:
class MyThread extends Thread {
  public void run() { System.out.println("Thread running"); }
}
new MyThread().start();
2. Implement Runnable interface (preferred):
class MyTask implements Runnable {
  public void run() { System.out.println("Task running"); }
}
new Thread(new MyTask()).start();

// With Lambda (modern way)
new Thread(() -> System.out.println("Lambda thread")).start();
Why Runnable is preferred: Java supports single inheritance; using Runnable keeps your class free to extend another class.
Q20Explain the Thread lifecycle (states).
NEW → Thread created but not started yet.
RUNNABLE → start() called; waiting for CPU or running.
BLOCKED → Waiting for a monitor lock (e.g., another thread holds synchronized block).
WAITING → Waiting indefinitely for another thread (wait(), join()).
TIMED_WAITING → Waiting for specified time (sleep(), wait(timeout)).
TERMINATED → run() method completed.
Q21What is synchronization? Why is it needed?
When multiple threads access shared data concurrently, race conditions can cause incorrect results. Synchronization ensures only one thread executes a critical section at a time.

synchronized keyword can be applied to methods or blocks. It uses the object's intrinsic lock (monitor).
class Counter {
  private int count = 0;
  
  public synchronized void increment() {
    count++; // thread-safe
  }
  
  // Or as a block (more granular):
  public void increment() {
    synchronized(this) { count++; }
  }
}
Q22What is a deadlock? How to prevent it?
Deadlock occurs when two or more threads are blocked forever, each waiting for a resource held by the other.

Prevention strategies:
1. Always acquire locks in the same order
2. Use tryLock() with timeout (from java.util.concurrent.locks)
3. Avoid holding multiple locks simultaneously
4. Use higher-level concurrency utilities (ExecutorService, etc.)

Swing Fundamentals

JFrame, JPanel, components, layouts

▾
Q23What is Swing? How does it differ from AWT?
AWT (Abstract Window Toolkit): Java's original GUI toolkit. Uses native OS components — looks different on each platform. Heavyweight components.

Swing: Built on top of AWT. Uses pure Java-drawn components (lightweight) — consistent look across platforms. Much richer component library. Part of javax.swing package.

JavaFX is the modern replacement for Swing with better styling (CSS), FXML, and 3D support.
Q24Name the key Swing components and their uses.
JFrame → Main application window
JPanel → Container for grouping components
JButton → Clickable button
JLabel → Non-editable text/image display
JTextField → Single-line text input
JTextArea → Multi-line text input
JComboBox → Dropdown menu
JCheckBox / JRadioButton → Selection options
JTable → Tabular data display
JMenuBar, JMenu, JMenuItem → Menus
Q25What are Layout Managers? Name and explain the main ones.
Layout Managers control how components are positioned in a container.

FlowLayout: Left to right, wraps on next line. Default for JPanel.
BorderLayout: 5 regions: NORTH, SOUTH, EAST, WEST, CENTER. Default for JFrame.
GridLayout: Equal-sized grid cells in rows and columns.
GridBagLayout: Most flexible. Components can span multiple cells. Complex to use.
BoxLayout: Single row or column.
CardLayout: Stack of panels, show one at a time (like tabs).
Q26How does Event Handling work in Swing?
Swing uses the Delegation Event Model. A component (event source) fires events; a listener handles them. 
JButton btn = new JButton("Click Me");

// Anonymous class (traditional)
btn.addActionListener(new ActionListener() {
  public void actionPerformed(ActionEvent e) {
    System.out.println("Button clicked!");
  }
});

// Lambda (modern, preferred)
btn.addActionListener(e -> System.out.println("Clicked!"));
Common listeners: ActionListener, MouseListener, KeyListener, WindowListener, ItemListener
Q27What is the Event Dispatch Thread (EDT)?
Swing is NOT thread-safe. All GUI updates must happen on the EDT (Event Dispatch Thread). Doing long tasks on the EDT freezes the UI.

SwingUtilities.invokeLater() → Schedules a Runnable to run on EDT (non-blocking).
SwingUtilities.invokeAndWait() → Waits until Runnable finishes on EDT (blocking).
SwingWorker → Runs background tasks off EDT and safely updates UI when done. 
SwingUtilities.invokeLater(() -> {
  JFrame frame = new JFrame("My App");
  frame.setVisible(true);
});

Exception Handling

try-catch-finally, checked vs unchecked

▾
Q28Checked vs Unchecked exceptions — what's the difference?
Checked exceptions: Caught or declared at compile time using throws. Subclass of Exception (excluding RuntimeException). Examples: IOException, SQLException.

Unchecked exceptions: Subclass of RuntimeException. Not required to handle at compile time. Examples: NullPointerException, ArrayIndexOutOfBoundsException, IllegalArgumentException.

Error: Serious JVM-level issues not meant to be caught: StackOverflowError, OutOfMemoryError.
Q29What is the purpose of the finally block?
finally block ALWAYS executes, whether exception occurs or not. Used for cleanup: closing file streams, DB connections, releasing resources.

Only exception: System.exit() prevents finally from running.

Try-with-resources (Java 7+): Automatically closes resources implementing AutoCloseable. Cleaner than explicit finally. 
// Try-with-resources
try (FileReader fr = new FileReader("file.txt");
     BufferedReader br = new BufferedReader(fr)) {
  String line = br.readLine();
} // fr and br auto-closed here
Q30How do you create a custom exception?
// Custom checked exception
class InsufficientFundsException extends Exception {
  private double amount;
  
  public InsufficientFundsException(double amount) {
    super("Insufficient funds: need " + amount + " more");
    this.amount = amount;
  }
  
  public double getAmount() { return amount; }
}

// Usage
public void withdraw(double amt) throws InsufficientFundsException {
  if (amt > balance)
    throw new InsufficientFundsException(amt - balance);
}

Maven, Logging & Deployment

Build tools, JAR/WAR, Tomcat, XML

▾
Q31What is Maven? What problem does it solve?
Maven is a build automation and dependency management tool. It solves:
1. Dependency management: Declare dependencies in pom.xml; Maven downloads them automatically from Maven Central
2. Build lifecycle: Standardized phases: validate → compile → test → package → install → deploy
3. Project structure: Convention over configuration

Key Maven commands:
mvn compile → compiles source
mvn test → runs tests
mvn package → creates JAR/WAR
mvn clean install → full build
Q32What is the difference between a JAR and WAR file?
JAR (Java Archive): Contains compiled Java classes and resources. Used for standalone applications or library packaging. Runnable with java -jar app.jar.

WAR (Web Archive): Contains web application (servlets, JSPs, HTML, web.xml). Deployed to a web server like Apache Tomcat or JBoss. Has specific directory structure (WEB-INF/, META-INF/).
Q33What logging frameworks have you used? How does logging work?
Common Java logging frameworks: java.util.logging (built-in), Log4j2, SLF4J (facade), Logback.

Log levels (low to high severity): TRACE → DEBUG → INFO → WARN → ERROR → FATAL

Best practice: Use SLF4J as the facade + Logback as the implementation. This lets you switch implementations without changing code. 
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

class MyService {
  private static final Logger log = 
    LoggerFactory.getLogger(MyService.class);
  
  void process() {
    log.info("Processing started");
    log.debug("Debug value: {}", someVar);
    log.error("Error occurred", exception);
  }
}
